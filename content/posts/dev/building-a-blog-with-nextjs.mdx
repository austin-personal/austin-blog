## 왜 직접 만들었나

기존 블로그 플랫폼도 좋지만, 개발자라면 직접 만드는 재미가 있다.
이 블로그는 **Next.js 15**의 App Router와 **MDX** 파일 기반으로 구축했다.

### 기술 스택

- **Framework**: Next.js 15 (Static Export)
- **Content**: MDX + gray-matter
- **Styling**: Tailwind CSS
- **Hosting**: GitHub Pages (무료!)

## MDX가 좋은 이유

MDX는 Markdown 안에서 React 컴포넌트를 사용할 수 있게 해준다.

```tsx
// MDX 안에서 이렇게 컴포넌트를 쓸 수 있다
<Callout type="info">
  이건 커스텀 컴포넌트입니다.
</Callout>
```

코드 하이라이팅도 `rehype-pretty-code`로 깔끔하게 처리된다.

## 정적 사이트의 장점

1. **빠른 로딩** — HTML 파일을 직접 서빙
2. **SEO 최적화** — 크롤러가 바로 읽을 수 있음
3. **무료 호스팅** — GitHub Pages로 비용 제로
4. **보안** — 서버가 없으니 공격 표면도 없음
